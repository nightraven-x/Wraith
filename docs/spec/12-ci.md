# Spec 12 — CI / GitHub Actions

> Research prerequisite: [../research/12-ci.md](../research/12-ci.md)
> Implements: `.github/workflows/build.yml`

---

## Purpose

Automate cross-compilation from Ubuntu (WSL-compatible) and produce release artifacts
on every `vX.Y.Z` tag push. Two jobs: `build` (always runs) and `installer` (tag only).

---

## Trigger

```yaml
on:
  push:
    tags: ['v*.*.*']
  workflow_dispatch:
```

`workflow_dispatch` allows manual runs for testing without pushing a tag.

---

## Job: build

Runs on: `ubuntu-latest`

**R1.** `actions/checkout@v4`

**R2.** `dtolnay/rust-toolchain@stable` with `targets: x86_64-pc-windows-gnu`

**R3.** `sudo apt-get install -y gcc-mingw-w64-x86-64`
Installs the MinGW cross-compiler and windres.

**R4.** `cargo build --release --target x86_64-pc-windows-gnu`
Produces `target/x86_64-pc-windows-gnu/release/wraith.exe`.

**R5.** Copy artifacts:
```bash
mkdir -p release
cp target/x86_64-pc-windows-gnu/release/wraith.exe release/
cp wraith.ini release/
```

**R6.** `actions/upload-artifact@v4` — name `wraith-windows-x64`, path `release/*`.

**R7.** On tag push only: `softprops/action-gh-release@v2` — upload `release/wraith.exe`
and `release/wraith.ini`. `fail_on_unmatched_files: false`.

---

## Job: installer

Runs on: `windows-latest`
Needs: `build`
Condition: `startsWith(github.ref, 'refs/tags/v')`

**R8.** `actions/checkout@v4`

**R9.** `actions/download-artifact@v4` — name `wraith-windows-x64`.
Downloads to `./` by default. Verify download path in research.

**R10.** Run `makensis installer\wraith.nsi` with `shell: cmd`.
`continue-on-error: true` — release proceeds even if installer build fails.

**R11.** `softprops/action-gh-release@v2` — upload `installer/wraith-setup.exe`.
`fail_on_unmatched_files: false` — handles case where makensis failed.

---

## Environment Variables

**R12.** `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}` — auto-provided by GitHub Actions.
No manual secret configuration needed.

---

## Full Workflow File

```yaml
name: Build & Release
on:
  push:
    tags: ['v*.*.*']
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: x86_64-pc-windows-gnu
      - run: sudo apt-get install -y gcc-mingw-w64-x86-64
      - run: cargo build --release --target x86_64-pc-windows-gnu
      - run: mkdir -p release && cp target/x86_64-pc-windows-gnu/release/wraith.exe release/ && cp wraith.ini release/
      - uses: actions/upload-artifact@v4
        with:
          name: wraith-windows-x64
          path: release/*
      - if: startsWith(github.ref, 'refs/tags/v')
        uses: softprops/action-gh-release@v2
        with:
          files: |
            release/wraith.exe
            release/wraith.ini
          fail_on_unmatched_files: false
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  installer:
    runs-on: windows-latest
    needs: build
    if: startsWith(github.ref, 'refs/tags/v')
    steps:
      - uses: actions/checkout@v4
      - uses: actions/download-artifact@v4
        with:
          name: wraith-windows-x64
      - run: makensis installer\wraith.nsi
        shell: cmd
        continue-on-error: true
      - uses: softprops/action-gh-release@v2
        with:
          files: installer/wraith-setup.exe
          fail_on_unmatched_files: false
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Tagging Convention

**R13.** Tags must be `vX.Y.Z` (e.g. `v1.0.0`). The update checker strips the leading
`v` and parses the rest as semver. Tags with suffixes (`v1.0.0-beta`) are valid git
tags but the installer job only runs on `v*.*.*` pattern — suffix tags won't trigger
the installer.

## Edge Cases

- **windres on ubuntu-latest:** Included in `gcc-mingw-w64-x86-64`. No separate install
  needed. Verify current package name in research.
- **makensis on windows-latest:** Pre-installed on GitHub-hosted Windows runners.
  Verify current version and availability in research.
- **Artifact download path in installer job:** `actions/download-artifact@v4` downloads
  to the current directory by default. `wraith.exe` and `wraith.ini` will be at `./`
  when `makensis` runs. The NSIS `File` directive must reference them from `./`.
- **Concurrent releases:** If two tags are pushed quickly, two workflows run
  independently. `softprops/action-gh-release@v2` creates separate releases per tag.
  No conflict.
- **build.rs on CI:** `build.rs` runs as part of `cargo build`. It invokes `windres`.
  If windres is installed after the toolchain step (which it is — apt-get runs after
  dtolnay/rust-toolchain), this is fine: `cargo build` runs last.
