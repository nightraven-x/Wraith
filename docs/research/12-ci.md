# Research 12 — CI / GitHub Actions

Verified facts for `.github/workflows/build.yml`.

---

## dtolnay/rust-toolchain

```yaml
- uses: dtolnay/rust-toolchain@stable
  with:
    targets: x86_64-pc-windows-gnu   # "targets" (plural), comma-separated string
```

- Input field: **`targets`** (NOT `target`) — confirmed from action README
- Accepts comma-separated string: `"x86_64-pc-windows-gnu"`
- Calls `rustup target add` internally — no separate step needed
- `@stable` resolves to current stable Rust — `OnceLock` (requires 1.70) is safe since stable is >1.70

## MinGW on ubuntu-latest

```yaml
- run: sudo apt-get install -y gcc-mingw-w64-x86-64
```

Installs and adds to PATH:
- `x86_64-w64-mingw32-gcc` — C compiler
- `x86_64-w64-mingw32-ar` — archiver
- `x86_64-w64-mingw32-windres` — resource compiler
- MinGW Windows headers (including `windows.h`) — sufficient for windres RC preprocessing
- All binaries in PATH for subsequent steps (no manual PATH addition needed)

## softprops/action-gh-release@v2

```yaml
- uses: softprops/action-gh-release@v2
  with:
    files: |
      release/wraith.exe
      release/wraith.ini
    fail_on_unmatched_files: false   # exact input name
```

- `files` input: pipe (`|`) YAML syntax for multiline — **NOT** `file: "...\n..."` literal `\n`
- Input name: **`fail_on_unmatched_files`** (not `fail_on_unmatched`)
- Creates release if it doesn't exist on tag push (published, not draft, by default)
- Multiple jobs CAN upload to the same release — second upload merges files
- For tag-triggered workflow: creates/updates release automatically

## GITHUB_TOKEN Permissions

```yaml
permissions:
  contents: write   # required for softprops/action-gh-release to create releases
```

Add at job level or workflow level. Default GITHUB_TOKEN may have `contents: read` only in some repo configurations — explicit `contents: write` is required.

## actions/download-artifact@v4

```yaml
- uses: actions/download-artifact@v4
  with:
    name: wraith-windows-x64
    # path: not specified → downloads to current working directory (repo root)
```

- Downloads to **current working directory** (`.`) by default, NOT a named subdirectory
- `wraith.exe` lands at `./wraith.exe` in the installer job workspace
- If NSIS `File "wraith.exe"` is run from repo root, it finds `./wraith.exe`

## makensis on windows-latest

- Pre-installed on `windows-latest` (Windows Server 2022)
- NOT pre-installed on `windows-server-2025` image
- Command: `makensis installer\wraith.nsi` — works in both cmd and PowerShell
- `continue-on-error: true` on installer step — allows release to succeed even if NSIS unavailable

## Workflow Trigger Pattern

```yaml
on:
  push:
    tags:
      - 'v*.*.*'    # matches v1.0.0, v1.0.1, v2.0.0 etc.
                    # does NOT match v1.0.0-beta (hyphen not covered by *.*.*)
  workflow_dispatch:
```

- Pattern `v*.*.*` is anchored to tags — does not match branch names
- `workflow_dispatch` (manual trigger): `startsWith(github.ref, 'refs/tags/v')` is FALSE → installer job skipped
- Installer job condition: `if: startsWith(github.ref, 'refs/tags/v')`

## Full Verified build.yml Structure

```yaml
name: Build & Release
on:
  push:
    tags: ['v*.*.*']
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: x86_64-pc-windows-gnu
      - run: sudo apt-get install -y gcc-mingw-w64-x86-64
      - run: cargo build --release --target x86_64-pc-windows-gnu
      - run: mkdir -p release && cp target/x86_64-pc-windows-gnu/release/wraith.exe release/ && cp wraith.ini release/
      - uses: actions/upload-artifact@v4
        with:
          name: wraith-windows-x64
          path: release/*
      - if: startsWith(github.ref, 'refs/tags/v')
        uses: softprops/action-gh-release@v2
        with:
          files: |
            release/wraith.exe
            release/wraith.ini
          fail_on_unmatched_files: false

  installer:
    runs-on: windows-latest
    needs: build
    if: startsWith(github.ref, 'refs/tags/v')
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/download-artifact@v4
        with:
          name: wraith-windows-x64
      - run: makensis installer\wraith.nsi
        continue-on-error: true
      - uses: softprops/action-gh-release@v2
        with:
          files: installer/wraith-setup.exe
          fail_on_unmatched_files: false
```
